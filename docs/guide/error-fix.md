# 错误修复

内置 23 种错误修复模式，自动诊断并快速修复常见问题。

## 修复能力概览

| 类别 | 模式数量 | 说明 |
|------|---------|------|
| 前端通用 | 12+ | 元素定位、样式问题、JS 错误等 |
| 后端 API | 4+ | 接口 404/500、参数错误等 |
| 数据库 | 4+ | 表缺失、列缺失、SQL 语法等 |
| 其他 | 3+ | 网络、配置、环境等 |

## 修复流程

```
错误捕获 → 模式匹配 → 根因分析 → 生成方案 → 应用修复 → 验证闭环
```

## 使用方式

### 自动诊断修复

```
这个页面报错了，帮我修复：
1. 打开 http://localhost:5173
2. 诊断错误
3. 自动修复
4. 验证修复结果
```

### 可用工具

| 工具 | 说明 |
|------|------|
| `browser_diagnose` | 自动错误诊断（根因 + 置信度） |
| `error_fix_suggestion` | 错误修复建议（23 种模式） |
| `browser_quick_fix` | 快速修复（多策略尝试） |
| `browser_verify_fix` | 修复验证闭环 |
| `fix_verify` | 修复结果验证 |
| `error_summary_md` | 错误摘要（Markdown） |

## 内置修复模式

### 前端模式

- 元素选择器失效 → 智能重定位
- CSS 变量缺失 → 自动补全
- JS 运行时错误 → 错误定位与建议
- 页面白屏 → 根因排查
- 加载缓慢 → 性能瓶颈分析

### 后端模式

- API 404 → 路由端点检查
- API 500 → 后端日志分析
- 参数错误 → 接口契约校验
- CORS 问题 → 跨域配置建议

### 数据库模式

- 表不存在 → 建表建议
- 列不存在 → ALTER TABLE 建议
- SQL 语法错误 → 语法修正
- 数据缺失 → 初始化脚本建议

## 注意事项

1. **备份优先** — 修复前建议备份数据和代码
2. **人工复核** — 自动修复建议仅供参考，生产环境需确认
3. **逐步验证** — 每次修复后运行验证，确认问题解决
4. **记录过程** — 所有修复操作都会记录在证据链中
