# 验证流程

AI-Verify MCP 支持完整的 7 阶段验证闭环，从环境准备到最终验证，全流程自动化。

## 7 阶段验证流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 环境准备  │ →  │ 2. 页面打开  │ →  │  3. 证据收集  │
└─────────────┘    └─────────────┘    └─────────────┘
                                                       ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  6. 报告生成  │ ←  │ 5. 问题修复  │ ←  │  4. 错误诊断  │
└─────────────┘    └─────────────┘    └─────────────┘
          ↓
    ┌─────────────┐
    │  7. 验证闭环  │
    └─────────────┘
```

## 阶段详解

### 阶段 1：环境准备

- 创建浏览器会话
- 配置视口大小
- 注入追踪器
- 设置网络拦截

### 阶段 2：页面打开

- 导航到目标 URL
- 等待页面加载完成
- 首屏截图
- 基础 DOM 快照

### 阶段 3：证据收集

- 全页截图
- DOM 结构快照
- Console 错误捕获
- 网络请求记录
- 无障碍扫描（a11y）
- 性能指标采集

### 阶段 4：错误诊断

- Console 错误聚合
- 网络失败请求分析
- 根因自动定位
- 置信度评估
- 修复模式匹配（23 种内置模式）

### 阶段 5：问题修复

- 自动生成修复建议
- 前端问题修复（CSS/JS/元素定位）
- 后端问题修复（API/数据库）
- 多策略尝试

### 阶段 6：报告生成

- JSON 结构化报告
- Markdown 可读报告
- HTML 可视化报告
- 证据链归档

### 阶段 7：验证闭环

- 修复后重新验证
- 对比修复前后差异
- 确认问题是否解决
- 输出最终结论

## 典型验证场景

### 场景 1：AI 生成页面验证

```
用户: "帮我验证这个登录页面"
  ↓
1. 打开页面 + 截图
2. 输入账号密码 + 点击登录
3. 检查 Console 错误 + 网络请求
4. 生成验证报告
  ↓
输出: 通过/失败 + 证据链
```

### 场景 2：视觉回归测试

```
用户: "对比一下这个页面和基线图"
  ↓
1. 加载基线截图
2. 当前页面截图
3. pixelmatch 像素对比
4. 生成差异高亮图
  ↓
输出: 差异像素数 + 对比图
```

### 场景 3：错误诊断修复

```
用户: "这个页面报错了，帮我看看"
  ↓
1. 打开页面复现问题
2. 收集 Console 错误和网络请求
3. 自动诊断根因
4. 生成修复建议
5. 应用修复 + 重新验证
  ↓
输出: 问题原因 + 修复方案 + 验证结果
```
