# 错误修复工具

内置 23 种错误修复模式，自动诊断 + 修复建议。

## 修复工具

| 工具 | 说明 |
|------|------|
| `browser_diagnose` | 自动错误诊断（根因分析 + 置信度） |
| `error_fix_suggestion` | 错误修复建议（23 种模式） |
| `browser_quick_fix` | 快速修复（多策略尝试） |
| `browser_verify_fix` | 修复验证闭环 |
| `fix_verify` | 修复结果验证 |
| `error_summary_md` | 错误摘要（Markdown） |
| `debug_investigate` | 深度调查分析 |

## browser_diagnose

自动诊断页面错误的根因。

**返回示例输出：**

```json
{
  "diagnosis": "页面 CSS 样式表加载失败",
  "root_cause": "CSS 文件路径错误导致 404",
  "confidence": 0.92,
  "category": "frontend",
  "suggestions": [
    {
      "action": "检查 CSS 文件路径",
      "details": "..."
    }
  ]
}
```

## error_fix_suggestion

根据错误信息匹配修复方案，内置 23 种模式。

**模式分类：

### 前端模式（12 种）

- 元素找不到 → 智能重定位
- CSS 变量缺失 → 自动补全建议
- JS 运行时错误 → 定位与建议
- 页面白屏 → 根因排查
- 加载缓慢 → 性能分析
- ...

### 后端模式（4 种）

- API 404 → 路由端点检查
- API 500 → 后端日志分析
- 参数错误 → 接口契约校验
- CORS 跨域 → 配置建议

### 数据库模式（4 种）

- 表不存在 → 建表建议
- 列不存在 → ALTER TABLE 建议
- SQL 语法错误 → 语法修正
- 数据缺失 → 初始化脚本建议

## browser_quick_fix

快速尝试修复，支持多种策略自动尝试。

**策略优先级：

1. 最常见原因优先
2. 最低风险操作优先
3. 可回滚能力优先

## 修复验证闭环

```
错误捕获
    ↓
模式匹配 → 置信度排序
    ↓
生成修复方案
    ↓
应用修复
    ↓
重新验证
    ↓
通过 → 完成
失败 → 尝试下一方案
```

## 注意事项

- 修复前建议备份
- 生产环境需人工复核
- 所有操作记录在证据链中
- 支持回滚能力
